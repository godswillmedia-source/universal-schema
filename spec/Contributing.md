Thank you for your interest in contributing to Universal Schema!

This project defines an open standard for AI agent communication. To keep the protocol clean, stable, and easy to implement, all contributions must follow the rules below.
	1.	Before submitting changes, open an Issue explaining:
	•	What you want to change
	•	Why it matters
	•	How it improves the protocol
	2.	Pull Requests must include:
	•	A clear description of the update
	•	Any breaking changes
	•	Example messages if applicable
	3.	All changes to the core specification (message envelope, required fields, versioning rules, capability format) must be backward-compatible unless explicitly marked for the next major version.
	4.	Extensions (new capabilities or optional fields) must:
	•	Not conflict with existing fields
	•	Not override core protocol behavior
	•	Follow naming formats defined in spec/capabilities.md
	5.	All contributors agree that their contributions are released under the CC BY 4.0 license of this repository.
	6.	Do not include production secrets, proprietary data, or private implementation details in Issues or PRs.
	7.	Large changes may require discussion or review from maintainers.

Thank you for helping improve Universal Schema. Your contributions shape the future of AI agent interoperability.
